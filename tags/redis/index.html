Redis 按 pattern 批量删除 key

DEL 不接受 pattern，要按前缀清一批 key 得绕一下。最常见是 EVAL 里用 KEYS 拿全集再分块 DEL：

redis-cli EVAL "local keys = redis.call('keys', ARGV[1])
for i=1,#keys,5000 do
  redis.call('del', unpack(keys, i, math.min(i+4999, #keys)))
end
return #keys" 0 'prefix:*'

分块是因为 Lua 的 unpack 在几千个参数后会撞 stack 上限，单次 DEL unpack(keys) 在大集合下会直接报错。

但 KEYS 本身是 O(N) 全表扫，会阻塞 Redis 主线程直到扫完整个 keyspace——线上库千万别这么干。生产环境用客户端侧 SCAN，游标式增量取，单次 DEL 控量：

redis-cli --scan --pattern 'prefix:*' | xargs -L 1000 redis-cli DEL

注意 KEYS / SCAN 的 pattern 是 glob（* ? [abc]），不是 regex。
