Task：YAML 写的 Make 替代，自带 sh 解释器

Task 是 Go 写的 task runner，单二进制 + YAML 配置，定位”现代 Make 替代”。同代工具里 Just 名气更大，但两者其实不在一条线上 —— Task 真正的差异点是自带 POSIX sh 解释器和原生增量 / 并行依赖，这两点决定了它能当轻量构建系统用，Just 不能。

Make 真痛点不在 Make 本身

GNU Make 三大平台都有，跨平台坑出在 Makefile 里每条命令依赖的宿主 shell 和 coreutils：

  • Shell 方言：/bin/sh 在 Ubuntu 是 dash、在老 macOS 是 bash、Alpine 是 ash，[[ ]] / 数组 / echo -e 行为各不一样。本机 macOS 跑通的 Makefile 一进 CI 就 silently 行为漂移
  • Coreutils 分裂：macOS 自带 BSD 版 sed / find / xargs，GNU 习惯的 sed -i 's/x/y/g' f 在 BSD 上必须写成 sed -i '' 's/x/y/g' f（-i 必须吃一个空字符串参数）；GNU 的 find -printf / xargs -r / date -d 在 BSD 上整个不存在
  • Tab vs 空格：Makefile 命令行必须 Tab 开头，VS Code 默认设置把 Tab 转空格 → missing separator 红一片

要让一个 Makefile 跨 macOS / Linux 都跑通，迟早塞进 ifeq ($(shell uname),Darwin) 分支，维护成本爆炸。

Task 解法：bundled sh + 原生 incremental

Task 内嵌 mvdan/sh，一个纯 Go 实现的 POSIX shell 解释器。for f in src/*.go; do echo $f; done 不再走 /bin/sh，macOS / Linux 行为一致，dash vs bash 的”在我机器上能跑”问题消失。

边界要写清楚：Task 只 bundle 了 shell 语法，没 bundle coreutils。rm / sed / find 仍然调系统的那份 —— BSD sed -i '' vs GNU sed -i 的坑 Task 不替你解决，要么显式 gsed，要么改写成只用 POSIX 公共子集。这一点官方文档也提：“any executables called must be available as a built-in or in the system’s PATH.”

增量构建走 sources + generates：

version: '3'

tasks:
  build:
    cmds:
      - go build -o bin/app ./cmd/app
    sources:
      - ./**/*.go
    generates:
      - bin/app

  test:
    deps: [build]          # deps 列表默认并行
    cmds:
      - go test ./...
  • sources 里所有文件算 checksum（可切 method: timestamp），下次跑发现指纹没变就跳过该 task —— 大项目 task build 没改代码时几毫秒返回
  • deps: 默认并行执行（前提是 deps 之间互不依赖），串行用 cmds: 里 - task: foo 调用
  • 配合 task --watch 监听文件变化自动重跑

跟 Just 的分野

Just 和 Task 经常被并列推荐，但 Just README 自己写得很清楚：“just is a command runner, not a build system, so it avoids much of make’s complexity and idiosyncrasies.” —— 故意不做依赖图也不做增量，任务每次从头跑。

维度JustTask
定位command runner（不是 build system）task runner + 轻量 build tool
Shell走宿主 shell自带 mvdan/sh，跨平台一致
依赖并行串行默认并行
增量跳过无sources + generates checksum
--watch无内置
语法自定义 DSL（精简 Makefile 风）YAML
参数传递位置参数一等公民：just deploy prod走变量：task deploy ENV=prod

判准：

  • 项目里命令独立、每次都该完整跑（lint、deploy、clear-cache） → Just，手感更轻
  • 任务有产物链、改一个文件只想重建受影响下游 → Task
  • 命令需要传位置参数（just deploy prod） → Just 直观
  • 团队里 Makefile 已经在用 sources / .PHONY / -j 这些 Make 特性，想替换 → Task 一对一替

装：brew install go-task。文档：https://taskfile.dev/。
