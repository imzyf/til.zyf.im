proto 装 Node.js 多版本

proto 是 moonrepo 团队 Rust 写的多 runtime 版本管理器，单二进制管 Node / Bun / Deno / Go / Rust 等。关键差异：shim-based 架构 + cd 自动切版本，不需要往 .zshrc / .bashrc 里挂任何 shell hook —— nvm / fnm 都做不到。

安装

bash <(curl -fsSL https://moonrepo.dev/install/proto.sh)

装到 ~/.proto/，shim 写到 ~/.proto/shims/ 并自动加进 PATH。不污染 /usr/local。

装 / 切 Node

proto install node lts            # 装最新 LTS
proto install node 20.10.0        # 装指定版本
proto pin node lts --to global    # 设全局默认
proto versions node               # 看本地装了哪些 + 远端可装版本

项目级版本：.prototools

一文件管所有 runtime，应该提交进 git（团队共享工具链版本，类比 composer.json 锁版本但管的是工具链本身）：

node = "20"
npm = "10"
bun = "1.3"

cd 进项目时无感切换 —— 触发机制是跑 shim 时读最近的 .prototools，不需要 shell hook，不需要 proto use。

--to 三档路径：

flag写到用途
--to local（默认）./.prototools项目级，提交进 git
--to user~/.prototools用户级，覆盖 global
--to global~/.proto/.prototools系统级默认

跟 nvm / fnm / mise 比

写法多 runtime自动切版本备注
nvmbash 函数❌ Node only需装 avn 等 hook每次 shell 启动加载，明显拖慢
fnmRust❌ Node only✅ 但需 shell hooknvm 平替，仍 Node-only
miseRust✅✅asdf 插件兼容，生态最大
protoRust✅✅ 无 hookWebAssembly 插件

最直接的对手是 mise（前身 rtx），asdf 插件兼容、社区大。proto 的差异是插件用 WASM 写（mise / asdf 用 shell / Lua 脚本），跨平台 + 沙盒化。两者都比 nvm / fnm 现代得多，没必要再用 nvm。按口味选：要生态用 mise，要插件干净 + 一律 shim 用 proto。

工作原理：shim 拦截

$ which node
~/.proto/shims/node                  ← 一个 shim 可执行

执行流程：node arg → shim 拦截 → 找最近 .prototools
       → 解析 node 版本 → exec ~/.proto/tools/node/<v>/bin/node

PATH 永远指向 ~/.proto/shims/，真二进制路径在每次调用时动态决定。这是为什么不需要 shell hook、不需要 proto use —— 切版本不动 PATH。代价是每次跑命令多一层 shim 解析，毫秒级，可忽略。

npx 跑 npm 包的几种姿势

npx 是 npm 自带工具，主要场景：跑本地 / 远端 / 指定版本的 npm 包，省掉全局安装。

跑本地 node_modules/.bin

项目装了 gulp 想跑：

node_modules/.bin/gulp -v
# 等价于
npx gulp -v

npx 查找顺序：node_modules/.bin → 系统 $PATH → 远端下载。所以系统命令也能 npx ls，但 shell builtin（如 cd）不行。

跑远端包，免全局安装

npx create-react-app my-app   # 临时下载 → 跑完即删
npx http-server               # 在当前目录起静态服务器

下次再跑还要重新下载。强制只用本地、不下载远端：npx --no-install <pkg>；反过来强制用远端忽略本地：npx --ignore-existing <pkg>。

指定 Node 版本跑脚本

npx node@0.12.8 -v
# v0.12.8

临时切版本比 nvm 省事。

-p 先装包再跑命令

npx -p node@0.12.8 node -v          # 先装 node@0.12.8，再跑 node -v
npx -p touch nodetouch gulpfile.js  # 先装 touch 包，再跑 nodetouch

跑 GitHub / Gist 源码

npx github:piuccio/cowsay hello
npx https://gist.github.com/zkat/4bc19503fe9e9309e2bfaa2c58074d32

要求远端必须是合法包：含 package.json 和入口脚本。
