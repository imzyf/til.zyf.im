Dockerfile 中三组易混指令怎么选

Dockerfile 里有几组指令名字相近、用途易混。按”什么时候执行 + 写到哪里”两条轴归位即可：

  • 构建时执行（写进 image 层）：RUN、COPY、ADD、ARG
  • 运行时执行（启动 container 时跑）：CMD、ENTRYPOINT
  • 构建期声明、运行时仍可见：ENV
  • 只是元数据（不执行也不影响行为）：LABEL

COPY vs ADD

两者都把构建上下文里的文件复制进 image，但 ADD 多了两个隐式行为：

  • 源是 .tar / .tar.gz 时自动解压到目标目录
  • 源是远程 URL 时自动下载

这两个 “智能” 行为正是 ADD 比 COPY 危险的地方——upstream tarball 内容变了，image 跟着变，且解压结果难审计。

口诀：默认用 COPY，确实需要解压本地归档时才上 ADD。远程文件用 RUN curl -O 更显式、更可控。

RUN vs CMD vs ENTRYPOINT

RUN 在构建时执行并提交一层。CMD 和 ENTRYPOINT 都只描述启动命令，区别在能不能被 docker run 的尾参覆盖：

指令何时跑默认参数？被尾参覆盖？
RUN构建时——
ENTRYPOINT启动时否不会（除非 --entrypoint）
CMD启动时是会

两种写法行为不同：

ENTRYPOINT ["/scripts/entrypoint.sh"]    # exec form
CMD "/scripts/start-fpm.sh"              # shell form

exec form（JSON 数组）= 直接 execve，进程是 PID 1，能正常收到 SIGTERM。 shell form（裸字符串）= Docker 包成 /bin/sh -c "..."，再整串作为参数传给 ENTRYPOINT，最终启动等价于：

/scripts/entrypoint.sh /bin/sh -c "/scripts/start-fpm.sh"

只有 CMD ["nginx"] 没 ENTRYPOINT 时，CMD 自己就是启动命令；docker run img bash 会把 nginx 整串替换成 bash。

口诀：主程序用 ENTRYPOINT exec form；可被覆盖的默认参数走 CMD exec form。两个都用 shell form 会丢掉信号转发，容器收不到 docker stop 的 SIGTERM，10 秒后被 SIGKILL 强杀。

LABEL vs ARG vs ENV

三者都是 “声明值”，但生命周期完全不同：

指令构建时可见？运行时可见？写进 image？
ARG是否否（但值会出现在 docker history，别塞密钥）
ENV是（声明之后的层）是（容器环境变量）是
LABEL否否是（仅元数据，docker inspect 可见）

ARG 还有一条作用域陷阱：FROM 之前声明的 ARG 只能在 FROM 行里展开，要在后续指令里用同名 ARG，得在 FROM 之后再声明一次：

ARG PHP_VERSION=8.3
FROM php:${PHP_VERSION}-fpm    # OK，这里能用

ARG PHP_VERSION                # 必须重新声明，下面才能用
RUN echo "Building PHP ${PHP_VERSION}"

口诀：版本号 / 镜像 tag 用 ARG；运行时配置（端口、路径、非密钥配置）用 ENV；作者 / VCS 地址 / 镜像描述用 LABEL。

Laradock 偷师：6 个值得抄走的 Docker 工程模式

Laradock 是 PHP/Laravel 的 docker-compose 全家桶模板。近年活跃度下降，新 Laravel 项目更多用 Sail。但它的 Dockerfile / compose 配置里有些经过多年打磨的模式，值得偷师——以下 6 条不限 PHP 场景。

1. PUID/PGID：容器用户对齐宿主，否则 bind mount 文件全是 root

ARG PUID=1000
ARG PGID=1000
RUN groupadd -g ${PGID} laradock && \
    useradd -l -u ${PUID} -g laradock -m laradock

容器默认 root 跑，写到 bind mount 的文件宿主端归 root 所有——开发者下次 vim 撞 permission denied，rm 要 sudo。把 host 的 id -u / id -g 通过 .env 注入 ARG，容器内建同 uid 用户，文件落地就是宿主用户。Linux dev 必踩；Mac/Win 因 Docker Desktop 自动做 uid 映射，不会感受到这个坑。

2. DEBIAN_FRONTEND=noninteractive 写 RUN 不写 ENV

# 不要：会污染容器运行时
ENV DEBIAN_FRONTEND=noninteractive

# 这样：只作用于该层 build
RUN DEBIAN_FRONTEND=noninteractive apt-get install -y tzdata

ENV 写进的是镜像层、运行时也生效——之后 docker exec 进容器手动装东西时，所有交互配置询问也被跳过（默认值生效），可能错过冲突选项或时区选择。RUN 内赋值只对那一条命令生效，构建期免提示、运行期保留交互。Docker 官方 Dockerfile best practices · ENV 同此建议。

3. extra_hosts: dockerhost 把”容器访问宿主”统一掉

extra_hosts:
  - "dockerhost:${DOCKER_HOST_IP}"
  # Linux 20.10+ 现在还可以这样：
  - "host.docker.internal:host-gateway"

Mac/Win Docker Desktop 自带 host.docker.internal，Linux 长期没有（20.10 才支持 host-gateway magic value）。Laradock 给容器塞统一别名 dockerhost——Xdebug 回连 IDE 时永远写 xdebug.client_host=dockerhost，跨平台共用一份 ini。

4. ARG INSTALL_XXX=false 的 opt-in 扩展矩阵

ARG INSTALL_XDEBUG=false
RUN if [ ${INSTALL_XDEBUG} = true ]; then \
    pecl install xdebug && docker-php-ext-enable xdebug; \
fi
WORKSPACE_INSTALL_XDEBUG=true
PHP_FPM_INSTALL_XDEBUG=true

一份 Dockerfile 适配 N 个项目：要 swoole 的项目 INSTALL_SWOOLE=true，不要的 false——不用为每个项目 fork 一份 Dockerfile。代价：build cache 粒度变粗（任一 ARG 变全层失效），高频切换的扩展（xdebug）值得拆独立 stage 减小重建范围。

5. docker-php-ext-disable 切扩展不重编译

# 生产关 xdebug，.so 文件仍保留
docker-php-ext-disable xdebug pcov

# 开发再 enable
docker-php-ext-enable xdebug

PHP 扩展本质是 .so + 一行 extension=xdebug.so ini 引用。docker-php-ext-disable 只移走 ini，.so 还在。开发/生产同一个镜像，不为”是否带 xdebug”重编两套——秒级切换。这条 official PHP image 自带，跟 Laradock 无关，但 Laradock 大量在用，是个被低估的工具。

6. db / cache 用命名 volume，不要 bind mount

services:
  mysql:
    volumes:
      - mysql:/var/lib/mysql            # 这样
      # - ./data/mysql:/var/lib/mysql   # 不要这样
volumes:
  mysql:

源码和数据卷分两种挂法 —— 源码要编辑器实时改，bind mount；数据要长期存活、和代码仓库脱钩，named volume。后者具体好在哪：

  • 误删难 —— rm -rf ./data 碰不到，得 docker volume rm 才删得了
  • 不污染 git —— bind mount 的 ./data/mysql/ 会进工作树，.gitignore 漏一次就把 ibdata 提进仓库
  • 备份 / 迁移有现成模式 —— named volume 跑一次 helper 容器就能打包；bind mount 还得另处理 owner
  • 权限不打架 —— 容器里 mysql uid 999 经常 chown 不动 host 目录；named volume 由容器自己初始化，没这层

规则：源码 bind mount，数据卷（mysql / postgres / redis / elasticsearch）走 named volume。
