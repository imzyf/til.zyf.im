tcpdump 速查

2020-06-29 #network

抓 HTTP 流量最常用的一行：

sudo tcpdump host httpforever.com and tcp port 80 -A -nn

完整文档 man tcpdump，下面只列高频项。

常用参数

参数作用
-i <iface>指定接口（不指定取第一个可用；-i any 监听所有）
-nn不解析主机名和端口名
-v / -vv / -vvv详细级别
-c <num>捕获指定数量后退出
-s <len>抓取长度（0 = 全包；新版默认 262144 bytes）
-w <file>写入 pcap 文件
-r <file>读取 pcap 文件离线解析
-A以 ASCII 打印 payload（截 HTTP：tcpdump -nSA port 80）
-X十六进制 + ASCII 打印（不要和 -A 一起用）
-S显示绝对序列号而不是相对编号
-tttt打印完整日期
-e显示以太网头（MAC 地址）

BPF 过滤语法

[proto] [src|dst] [host|net|port] <对象> [逻辑运算]

tcp and dst port 80 and src net 10.0.0.0/8
  • 协议：ip tcp udp icmp arp vlan ip6 …
  • 方向：src dst src or dst src and dst
  • 对象：host 10.0.0.1 / net 192.168.0.0/16 / port 443
  • 逻辑：and (&&) / or (||) / not (!) / ( )
sudo tcpdump -i en0 host 192.168.50.1          # 源或目的
sudo tcpdump -i en0 src host 192.168.50.1      # 源
sudo tcpdump -i en0 dst host 192.168.50.1      # 目的
sudo tcpdump -i en0 port 8080                  # 端口
sudo tcpdump -i en0 net 192.168                # 网段
sudo tcpdump -i en0 tcp                        # 协议

输出格式

14:24:12.196582 IP 192.168.91.29.56930 > 146.190.62.39.http: Flags [SEW], seq 193726297, win 65535, options [...], length 0

时间戳 / 协议层 / 源.端口 > 目标.端口 / TCP 标志位 / seq ack win 等字段 / 报文长度。

TCP Flags

TCP Flag缩写含义
SYNS会话建立请求
ACKA确认发送方的数据
FINF终止标识
RESETR立即中止连接
PUSHP立即推送数据
URGENTU优先于其他数据
NONE.占位符，通常用于 ACK

参考 tcpdump Flags。

接口列表

sudo tcpdump -D
# 1.en0 [Up, Running]
# 2.p2p0 [Up, Running]
# ...

macOS 下监听适配器必须 root。

实例

抓 eth1 上目的地址是 192.168.1.254 或 192.168.1.200、端口 80 的 TCP：

sudo tcpdump -i eth1 '((tcp) and (port 80) and ((dst host 192.168.1.254) or (dst host 192.168.1.200)))'

只抓 SYN：

sudo tcpdump -i eth1 'tcp[tcpflags] = tcp-syn'

抓 SYN + ACK：

sudo tcpdump -i eth1 'tcp[tcpflags] & tcp-syn != 0 and tcp[tcpflags] & tcp-ack != 0'

抓 DNS 请求：

sudo tcpdump -i en0 udp dst port 53

流量大时用 -c 限制数量：

sudo time tcpdump -nn -i en0 'tcp[tcpflags] = tcp-syn' -c 10000 > /dev/null

实时抓 8000 端口的 GET 包写入文件：

sudo tcpdump -i eth0 '((port 8000) and (tcp[(tcp[12]>>2):4]=0x47455420))' -nnAl -w /tmp/GET.log
