git-extras：把多步 git 操作压成一个动词

2026-05-08 #git

tj/git-extras 是 70+ 个 git-xxx 子命令的合集，brew install git-extras 装完直接以 git xxx 调用。70 个里真正值得放进肌肉记忆的，是那些把多步 git incantation 压成一个动词的：

git summary 是接手陌生仓库的瑞士军刀，一行看到全貌：

$ git summary
 project     : blog-hugo
 repo age    : 1 year, 1 month
 commits     : 37
 active on   : 19 days
 authors     :
    36 zhaoyifan   97.3%

git effort <path> 接着追问”哪些文件改得最多”，按 commits × active days 排，找重构 / 归档候选最直接：

$ git effort --above 3 content/posts/
  path                                      commits  active days
  hugo.yaml..............................   10       6
  README.md..............................    7       5
  • git undo [n] — 退掉最近 N 个 commit（保留改动），不用再想 --soft HEAD~n 的对错。
  • git rename-branch new-name — 同时 rename local + remote，省掉 branch -m、push origin :old、push -u origin new 三步。
  • git delete-merged-branches — 一条命令清掉所有已合并到当前分支的本地分支。
  • git ignore "*.log" build/ — 命令行追加 .gitignore，不用开文件。

zsh 补全要单独 source：

echo 'source /opt/homebrew/opt/git-extras/share/git-extras/git-extras-completion.zsh' >> ~/.zshrc

剩下 60+ 命令按需查 https://github.com/tj/git-extras/blob/master/Commands.md。
