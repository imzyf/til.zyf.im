MySQL 客户端提示符自定义

2016-07-31 #mysql

默认的 mysql> 提示符不显示当前用户 / host / 数据库——切了几次连接后 DROP TABLE 错库是真实事故。把提示符换成 (user@host) [db]> 是 5 秒杜绝这类事故的方式。

四种设法，作用域和反斜杠规则不同：

方法作用域反斜杠
mysql --prompt="..." CLI仅当前连接单 \
连接后输入 prompt ...仅当前连接单 \
MYSQL_PS1 环境变量当前 shell 所有连接单 \
~/.my.cnf 的 [mysql] 段当前用户所有连接双 \\

推荐永久方案落 ~/.my.cnf：

[mysql]
prompt="(\\u@\\h) [\\d]>\\_ "

效果：(root@127.0.0.1) [mydb]> 。

为什么 my.cnf 必须双反斜杠

option 文件解析器自己有一套反斜杠转义（\s → 空格、\n → 换行、\t → tab），跟 prompt 转义序列正面冲撞。写单 \s 会被 option 解析层先吃掉变成空格，根本到不了 prompt 解析层，秒数字段就没了。双 \\ 是先把 option 层逃掉，让真正的 \s 落到 prompt 层。

其他三种方式（CLI / 交互 prompt / 环境变量）不经 option 解析，单 \ 即可。混淆这条是从配置文件抄到命令行（或反过来）调不通的高频原因。

常用转义序列

  • \u 当前用户 / \h 服务器 / \d 当前数据库
  • \U 完整 user@host
  • \D 完整日期 / \R 24 时制 / \m 分 / \s 秒
  • \T 在事务块中显示 *（活在未提交事务里很容易忘）
  • \c 自增的语句计数器
  • \_ 一个空格（结尾留空白比裸空格稳）

完整列表见 MySQL 8.4 mysql Client Commands。
