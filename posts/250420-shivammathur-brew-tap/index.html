macOS 多版本 PHP：shivammathur brew tap

2024-05-20 #php#macos

Homebrew core 只供应当前 stable 一个 PHP（当下是 8.5），多项目共存——老 Laravel 跑 7.4、新项目用 8.3——就走不通，必须从 shivammathur/php tap 拉版本化 formula。覆盖 5.6 → 8.6，每个主版本另带 -debug / -zts / -debug-zts 三个变体。

brew tap shivammathur/php
brew install php@7.4 php@8.3
brew unlink php@7.4 && brew link --overwrite php@8.3   # 切换默认
/opt/homebrew/opt/php@7.4/bin/php -v                   # 或走绝对路径免 link

C 扩展（imagick / redis / xdebug …）配套用 shivammathur/extensions tap。命名是「扩展 × PHP 版本」笛卡尔积——每个组合一个独立 formula，PHP 版本必须随扩展名一起写出，所以多版本同装 N 个扩展就是 N×M 个 bottle。

brew tap shivammathur/extensions
brew install shivammathur/extensions/imagick@8.3
brew install shivammathur/extensions/xdebug@7.4

为什么不直接 pie install imagick/imagick / pecl install imagick：macOS 上自编扩展要先把 imagemagick / phpize / autoconf / OpenSSL header 这套链拉齐，ImageMagick 6 与 7 不兼容、oniguruma 之类间接依赖踩坑率高。brew bottle 直接装预编译产物，把 C 工具链问题外包给 tap 维护者；代价是扩展和某个 php@x.y 强绑死，切 PHP 版本得相应换扩展 formula。

只装一个 PHP 留 core 的 php 就够，多版本 + 扩展才进 tap。
