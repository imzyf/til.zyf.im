PHP CLI 排查速查：`--ri` / `--re` / `php -a` 常被忽略

2020-04-15 #php

php 的 CLI 选项里有几个 flag 大多数人不知道，能省一堆 phpinfo() | grep。

php --ri <extension> —— 看某个扩展的当前有效配置（合并所有 ini 之后的运行时值，不是源 ini 文件内容）：

php --ri gd
php --ri opcache

比 php -i | grep -A 10 gd 准 —— --ri 给的是 PHP 真正认到的值。

php --re <extension> —— 反射列出扩展提供的所有函数 / 类 / 常量，顺便确认装没装：

php --re decimal
# 没装时输出：Exception: Extension decimal does not exist

php -a —— REPL 交互模式，带 tab 补全（函数 / 类 / 常量名），比 php -r '...' 拼字符串方便：

php -a
php > date('Y-m-d', 1234567890);

需要 PHP build 时带 readline，Homebrew 装的 php 默认有。

php-fpm -tt —— FPM 配置语法检查 + dump 解析后的最终值（双 t，单 t 只验语法不 dump）：

php-fpm -tt 2>&1 | grep memory_limit

其他常用一行：

php --ini                                                  # ini 文件路径
php -i | grep -E "Loaded Configuration|Scan this dir"      # ini 加载顺序
php -m                                                     # 已加载模块
php -r "echo ini_get('memory_limit');"                     # 看任意 ini 值
php -r "phpinfo();" | grep configure                       # 编译参数
php -r "print_r(openssl_get_cert_locations());"            # openssl cert 路径
