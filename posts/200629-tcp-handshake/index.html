TCP 三次握手 / 四次挥手

2020-06-29 #network#tcp

客户端 A、服务器 B，初始序号 seq，确认号 ack。

三次握手（建立连接）

初始：B 监听，A 打开。

  • A → B：seq = x（A 发连接请求，进入 SYN-SENT）
  • B → A：ack = x+1, seq = y（B 确认，进入 SYN-RCVD）
  • A → B：ack = y+1（A 再确认，进入 ESTABLISHED）

B 收到 A 的确认后也进入 ESTABLISHED。

为什么三次？ 防止失效的连接请求突然传到服务器，让服务器误以为要建连接。

四次挥手（释放连接）

  • A → B：seq = u（A 发释放报文，进入 FIN-WAIT-1）
  • B → A：ack = u+1, seq = v（B 确认，半关闭，B 还能发数据，进入 CLOSE-WAIT）
  • B → A：ack = u+1, seq = w（B 重发确认号，进入 LAST-ACK）
  • A → B：ack = w+1, seq = u+1（A 确认，进入 TIME-WAIT）

经过 2MSL 后 A 才进入 CLOSED。

为什么 TIME-WAIT 要等 2MSL？

  • 保证 A 发的最后一个 ACK 能到达 B（否则 B 重传 FIN）
  • 让本次连接的失效报文在网络中消失，不污染下一次连接

抓包验证

sudo tcpdump -i en0 host www.qq.com and tcp -S -c 50
