httpbingo.org —— 按需返回任意 HTTP 响应

2026-05-26 #network

写 HTTP 客户端时最难造的是失败：500、慢响应、3xx 链、basic auth 401。靠真实业务 API 不可控也污染数据。httpbingo.org 就是干这个——按 URL 告诉它要什么响应，它就回什么。

curl https://httpbingo.org/status/503         # 强制 503
curl https://httpbingo.org/delay/5            # 5 秒后响应（测超时）
curl https://httpbingo.org/redirect/3         # 3 次 302 链（测 follow 上限）
curl -u user:pass https://httpbingo.org/basic-auth/user/pass
curl https://httpbingo.org/anything           # 把请求 headers / body / query 原样 echo 回来

/anything 是 debug 神器——客户端实际发了什么 header、是不是按预期带了 cookie、proxy 有没有偷改 body，一行 curl 看到。

域名容易和老前辈 httpbin.org 混。httpbingo.org 是 mccutchen/go-httpbin 跑的公网实例，Kenneth Reitz 那个老 httpbin 的 Go 重写。老版自 v0.6.1（2017-08）后没再发版，README 还挂着 travis-ci.org 的 badge（travis 早已对 OSS 停服）。go-httpbin 零依赖、stdlib only、持续发版（v2.22 / 2026），多了 /sse /jsonl /websocket/echo，少了 /brotli（Go stdlib 不带）。

CI 里别直接打公网

免费服务限流 / 下线一次就 red 一片。CI 里起个 sidecar：

docker run -p 8080:8080 ghcr.io/mccutchen/go-httpbin

写 Go 的话还能直接当 library 嵌进单测起 httptest.Server，连 sidecar 都省了。
