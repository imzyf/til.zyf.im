折叠生成文件：linguist-generated vs gitlab-generated

2026-06-03 #github

生成文件（dist/、*.min.js、lock 文件）有两个麻烦：把仓库语言占比统计带歪，把 PR diff 撑出几千行。两个平台都在 .gitattributes 里解决，但机制差别比想象大——GitHub 一个 attribute 全包，GitLab 按用途拆成三套，标错只生效一半。

GitHub：linguist-* 一个搞定两件事

GitHub 的语言统计和 diff 折叠都归 Linguist 一套管，所以 linguist-generated 一标同时踢出语言统计 + 折叠 diff：

dist/* linguist-generated
package-lock.json -linguist-generated   # 前导 - = 反向排除

同家族还有 linguist-vendored（第三方库）、linguist-documentation（文档）、linguist-language=PHP（改判语言）。判准：generated = 机器产物，vendored = 别人的代码，都被踢出统计，区别只在 diff 折不折叠。

attribute不计统计diff 折叠
linguist-generated✅✅
linguist-vendored✅❌
linguist-documentation✅❌

GitLab：三套各管一摊

GitLab 只有两个 gitlab- 专属 attribute，各自只干一件事；语言统计反而借用 GitHub 的 linguist-*——它的统计引擎 go-enry 是 Linguist 的 Go 移植，认 linguist- 前缀。

用途GitLab 用什么
MR diff 折叠gitlab-generated（专属）
语法高亮改判gitlab-language=prolog（专属，=text 关高亮）
语言统计排除/改判linguist-generated / linguist-vendored / …

最大的坑：在 GitLab 标 gitlab-generated 只折叠了 diff，文件照样算进语言占比；想同时从统计里拿掉，得再补一条 linguist-generated。和 GitHub 一个 attribute 全包正相反。

*.min.js gitlab-generated      # 只折叠 diff
*.min.js linguist-generated    # 才会从语言统计条消失

另注：GitLab 会自动识别部分生成文件（protobuf 产物、go generate 的 Go 文件、source map），不标也可能折叠，这也是”有的折有的没折”不一致反馈的来源（#572763）。

共同点：都是 .gitattributes 的 override

linguist-* / gitlab-* 和 LFS 的 filter=lfs、文件锁 lockable、行尾 text=auto 堆在同一个文件。记法：git 原生 attribute（filter / text / merge）各平台通用；带平台前缀的只在对应平台网页生效，换到本地仓库一律失效。

See Linguist overrides / GitLab Git attributes。
