git commit --fixup 配 rebase autosquash 收拢历史

2026-06-03 #git

code review 改意见时，最容易留下 fix typo、address review 这种噪音 commit。--fixup + autosquash 让每个补丁自动并回它对应的原始 commit，历史只剩有意义的那几个。

发现 commit A 要补个改动时，不要新开一个独立 commit，而是：

git commit --fixup A      # message 自动写成 "fixup! <A 的标题>"

A 是占位符，换成目标 commit 的任意 commit-ish:hash(abc1234)、HEAD~2、分支名，或 :/登录 模糊匹配 message。下文 A / B 同理，只是示意图里给 commit 起的标签。

之后一把 rebase 收拢：

git rebase -i A^

打开 todo 时 git 已经自动把 fixup! 那条挪到 A 正下方、并标好 fixup 动作，直接存盘退出即可吃进 A:

pick   A   feat: 登录接口
fixup  A'  fixup! feat: 登录接口   ← 自动排序 + 自动标记
pick   B   feat: 注册接口

~/.gitconfig 里这行是关键——它让 rebase -i 默认就开自动排序，否则每次得手动加 --autosquash,或在编辑器里自己挪行：

[rebase]
  autosquash = true

两个易混点：

  • --fixup vs --squash:fixup! 丢弃补丁的 message，完全用原 commit 的;squash! 把两条 message 拼起来让你编辑。改 bug 用 fixup，要补充说明用 squash。
  • autosquash 只改 -i 的默认行为,不是全自动——仍要你手动触发一次 git rebase -i,它只负责把 fixup!/squash! 排到位。

--fixup 还能直接指向”动了哪个文件就修哪个 commit”:

git commit --fixup :/登录    # 模糊匹配最近 message 含"登录"的 commit

See man git-rebase → --autosquash,man git-commit → --fixup。
