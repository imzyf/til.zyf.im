2026-05-09 #network

CORS：Allow-Origin 为 * 时不能搭配 credentials

带 cookie 的跨域请求（fetch(..., { credentials: 'include' })、xhr.withCredentials = true）要求 Access-Control-Allow-Origin 是具体 origin，不能是 *。Firefox 报：

Credential is not supported if the CORS header ‘Access-Control-Allow-Origin’ is ’*’

Chrome 报：

The value of the ‘Access-Control-Allow-Origin’ header in the response must not be the wildcard ’*’ when the request’s credentials mode is ‘include’

服务端读请求头 Origin，校验白名单后回写具体值：

Access-Control-Allow-Origin: https://app.example.com
Access-Control-Allow-Credentials: true
Vary: Origin

Vary: Origin 防止 CDN / 中间层把 A 域名的响应缓存给 B 域名。

cloc / tokei / scc：codebase LOC 统计的三代工具

数代码行数（LOC by language）这件事有三代工具，每代换语言、换取舍：

cloctokeiscc
实现 / 起年Perl / 2006Rust / 2016Go / 2018
Linux kernel 用时~64s~1.4s~0.8s
默认并行❌（要 --processes=N）✓✓
安装brew install clocbrew install tokeibrew install scc
独有能力--diff 对比两个 commit；读 .zip / .tarJSON / YAML 输出复杂度 / COCOMO / 重复检测

scc 那三个名词各自的意思：

  • 复杂度：数代码里 if / for / && 这类分支关键字，估算 cyclomatic complexity。
  • COCOMO：1981 年 Barry Boehm 提出的成本模型，给 KLOC 估”工业平均要多少人月、多少钱”。
  • 重复检测：按文件 hash 判定，相同内容只算一次（monorepo / vendor 目录里常见，cloc / tokei 会重复计入）。

对同一个目录（这个 blog 的 til/src/）跑默认命令，差异自己跳出来：

$ cloc til/src/
Language           files          blank        comment           code
Astro                 10            117             62           1340
TypeScript             7             14             49             89
SUM:                  17            131            111           1429

$ tokei til/src/
 Language         Files        Lines         Code     Comments       Blanks
 Astro               10         1519         1344           59          116
 TypeScript           7          152           97           44           11
 Total               17         1671         1441          103          127

$ scc til/src/
Language        Files       Lines    Blanks  Comments       Code Complexity
Astro              10       1,519        99        49      1,371         55
TypeScript          7         152        11        44         97          2
Total              17       1,671       110        93      1,468         57
Estimated Cost to Develop (organic) $40,425
Estimated Schedule Effort (organic) 4.06 months
Estimated People Required (organic) 0.88

cloc 没有 Lines 总数列（自己加 blank+comment+code）；tokei 加了 Lines；scc 多 Complexity 列 + 一段 COCOMO 估算块。同样的 17 个文件，三个工具给出三个 Code 总数：1429 / 1441 / 1468——blank 和 comment 怎么判，每家略不同。

cloc 的 --diff 是至今没被替代的核心——code review 想量化”这个 PR 改了多少 code 行”时，tokei / scc 都做不到：

$ cloc --git --diff HEAD~5 HEAD
Language        files          blank        comment           code
Markdown
 same              0              0              2            924
 modified          2              0              0              9
 added             0              0              0              4
 removed           0              0              0             19
Bourne Shell
 same              0              0              5             14
 modified          1              0              0              1
 added             0              0              0              1
 removed           0              0              0              0
SUM:
 same              0              0              7            938
 modified          3              0              0             10
 added             0              0              0              5
 removed           0              0              0             19

按语言分组，每组拆 same / modified / added / removed；想细到文件级加 --by-file。比 git diff --stat 信噪比高一截。

怎么选： 默认装 scc（最快 + 复杂度）；CI 输出要 JSON 接管线用 tokei；要 git diff 的 LOC 变化才掏 cloc。

2026-05-08 #php

PHP 安装工具的几条路径

PHP 安装按”装什么 + 在哪装”分四条路径，每条各自的工具：

纯 PHP 代码（复制即用）

  • PEAR — PHP Extension and Application Repository。最早的类库分发，用前要 include。已被 Composer 取代，基本退场。
  • Composer — PHP 依赖管理工具，灵感来自 npm / Bundler。composer.json 声明依赖，composer install 拉齐，事实标准。

C 扩展，裸机环境（编译 + 改 php.ini）

  • PECL — PHP Extension Community Library。PEAR 的子项目，分发 redis、apcu 这类底层扩展。已被官方标记 deprecated。
  • PIE — The PHP Installer for Extensions。PHP 基金会官方接管，取代 PECL，命令风格抄 Composer。

C 扩展，Docker 容器场景

  • docker-php-ext-install — 官方 PHP Docker 镜像内置脚本，容器里装 C 扩展的事实标准。
  • install-php-extensions — 上面的友好封装，自动处理依赖、编译选项、共享库。Dockerfile 里写 PHP 几乎人人用它。

PHAR 形式的 CLI 开发工具（PHPUnit、phpstan、psalm 这种）

  • PHIVE — PHAR Installation and Verification Environment。装 PHAR 工具并校验签名，避免污染 composer.json。

七个命令并排看：

# 纯 PHP 代码
pear install Mail                        # 老路子，已退场
composer require monolog/monolog         # 现代

# C 扩展（裸机）
pecl install apcu                        # 老路子（deprecated）
pie install apcu/apcu                    # 现代

# C 扩展（Docker），以装 gd 为例——需要系统库 + configure
apt-get install -y libfreetype-dev libjpeg-dev libpng-dev \
  && docker-php-ext-configure gd --with-freetype --with-jpeg \
  && docker-php-ext-install gd            # 官方：3 步联动
install-php-extensions gd                 # 友好封装：1 行搞定

# PHAR CLI 工具
phive install phpunit

记忆：

  • PHP 代码 → Composer
  • C 扩展，裸机 → PIE（PECL 已 deprecated）
  • C 扩展，Docker → install-php-extensions
  • PHAR CLI 工具 → PHIVE

2022-07-26 #node

npx 跑 npm 包的几种姿势

npx 是 npm 自带工具，主要场景：跑本地 / 远端 / 指定版本的 npm 包，省掉全局安装。

跑本地 node_modules/.bin

项目装了 gulp 想跑：

node_modules/.bin/gulp -v
# 等价于
npx gulp -v

npx 查找顺序：node_modules/.bin → 系统 $PATH → 远端下载。所以系统命令也能 npx ls，但 shell builtin（如 cd）不行。

跑远端包，免全局安装

npx create-react-app my-app   # 临时下载 → 跑完即删
npx http-server               # 在当前目录起静态服务器

下次再跑还要重新下载。强制只用本地、不下载远端：npx --no-install <pkg>；反过来强制用远端忽略本地：npx --ignore-existing <pkg>。

指定 Node 版本跑脚本

npx node@0.12.8 -v
# v0.12.8

临时切版本比 nvm 省事。

-p 先装包再跑命令

npx -p node@0.12.8 node -v          # 先装 node@0.12.8，再跑 node -v
npx -p touch nodetouch gulpfile.js  # 先装 touch 包，再跑 nodetouch

跑 GitHub / Gist 源码

npx github:piuccio/cowsay hello
npx https://gist.github.com/zkat/4bc19503fe9e9309e2bfaa2c58074d32

要求远端必须是合法包：含 package.json 和入口脚本。

来源：npx 使用教程 - 阮一峰

proto 装 Node.js 多版本

2021-09-14 #hive

Hive OVER (PARTITION BY) 窗口函数

业务上要算”当年累计 income”这种基于组的滚动指标，普通 SUM(income) GROUP BY ... 一组只剩一行，行级累计要靠开窗函数 OVER (PARTITION BY ...)：每行都返回一个基于”窗口”内的聚合值，原始行数不变。

CREATE TABLE default.test_over_partition (
  fdate     Date,
  year      Int,
  month     Int,
  category1 String,
  category2 String,
  income    Double
);

INSERT INTO default.test_over_partition VALUES
  ('2020-01-01', 2020, 1, '3C', '电脑', 1010),
  ('2020-01-01', 2020, 1, '3C', '手机', 1011),
  ('2020-02-01', 2020, 2, '3C', '电脑', 1012),
  ('2020-02-01', 2020, 2, '3C', '手机', 1013),
  ('2020-03-01', 2020, 3, '3C', '电脑', 1014),
  ('2020-03-01', 2020, 3, '3C', '手机', 1015),
  ('2021-04-01', 2021, 4, '3C', '电脑', 1016),
  ('2021-04-01', 2021, 4, '3C', '手机', 1017),
  ('2021-05-01', 2021, 5, '3C', '电脑', 1018),
  ('2021-05-01', 2021, 5, '3C', '手机', 1019);

-- 按 year × category 分组，按 fdate 排序，行级累计 income
SELECT fdate, year, category2, income,
       SUM(income) OVER (PARTITION BY year, category1, category2 ORDER BY fdate) AS ttl_year_income
FROM default.test_over_partition;

PARTITION BY 决定窗口边界（每个 year × category 一个窗口），ORDER BY 决定窗口内累计的方向。换 AVG / RANK / ROW_NUMBER 等其他分析函数，套路一样。

2020-09-11 #laravel

脱离 Laravel 跑 illuminate/validation

illuminate 组件都是独立 composer 包，illuminate/validation 在非 Laravel 项目里也能跑——手动把 FileLoader → Translator → Factory 三件套接起来即可。

安装

composer require illuminate/validation illuminate/translation illuminate/filesystem

illuminate/filesystem 不是 illuminate/validation 的依赖，用 FileLoader 必须单独装。

准备语言文件

从 Laravel-Lang/lang 复制要用的语言文件到本地，例如 lang/zh-CN/validation.php。

装配

use Illuminate\Filesystem\Filesystem;
use Illuminate\Translation\FileLoader;
use Illuminate\Translation\Translator;
use Illuminate\Validation\Factory;

$loader = new FileLoader(new Filesystem(), __DIR__ . '/../lang');
$translator = new Translator($loader, 'zh-CN');
$validator = new Factory($translator);

用

$v = $validator->make(
    ['name' => ''],
    ['name' => 'required|string|min:2'],
);

if ($v->fails()) {
    print_r($v->errors()->all());
}

完整规则参考 Laravel 文档的 Manually Creating Validators。

2020-08-14 #mysql

MySQL JSON 列建索引

MySQL 5.7+ 支持原生 JSON 数据类型。两个坑：

  • JSON 列不能有 non-NULL 默认值——要么 nullable，要么应用层填
  • 不能直接在 JSON 字段上建索引——要靠 Generated Column 包一层

用 Generated Column 索引 JSON 字段

把要查的 JSON 路径抽成一个虚拟列，再在虚拟列上建索引：

CREATE TABLE players (
  id               INT UNSIGNED NOT NULL,
  player_and_games JSON NOT NULL,
  name_virtual     VARCHAR(20) GENERATED ALWAYS AS (player_and_games ->> '$.name') NOT NULL,
  PRIMARY KEY (id),
  KEY idx_name (name_virtual)
);

->> 是 -> + JSON_UNQUOTE 的简写——直接拿字符串而不是 quoted JSON value。

Virtual 列只存元数据不落盘（默认），Stored 列落盘到磁盘。索引 JSON 字段几乎都用 Virtual，省空间且没性能差。

验证索引命中

EXPLAIN SELECT * FROM players WHERE name_virtual = 'Sally';
-- key: idx_name

参考 MySQL 5.7 新特性 JSON · 如何索引 JSON 字段 | mysql.taobao.org · JSON 函数参考

2020-06-29 #network#tcp

TCP 三次握手 / 四次挥手

客户端 A、服务器 B，初始序号 seq，确认号 ack。

三次握手（建立连接）

初始：B 监听，A 打开。

  • A → B：seq = x（A 发连接请求，进入 SYN-SENT）
  • B → A：ack = x+1, seq = y（B 确认，进入 SYN-RCVD）
  • A → B：ack = y+1（A 再确认，进入 ESTABLISHED）

B 收到 A 的确认后也进入 ESTABLISHED。

为什么三次？ 防止失效的连接请求突然传到服务器，让服务器误以为要建连接。

四次挥手（释放连接）

  • A → B：seq = u（A 发释放报文，进入 FIN-WAIT-1）
  • B → A：ack = u+1, seq = v（B 确认，半关闭，B 还能发数据，进入 CLOSE-WAIT）
  • B → A：ack = u+1, seq = w（B 重发确认号，进入 LAST-ACK）
  • A → B：ack = w+1, seq = u+1（A 确认，进入 TIME-WAIT）

经过 2MSL 后 A 才进入 CLOSED。

为什么 TIME-WAIT 要等 2MSL？

  • 保证 A 发的最后一个 ACK 能到达 B（否则 B 重传 FIN）
  • 让本次连接的失效报文在网络中消失，不污染下一次连接

抓包验证

sudo tcpdump -i en0 host www.qq.com and tcp -S -c 50

2020-06-29 #network

tcpdump 速查

抓 HTTP 流量最常用的一行：

sudo tcpdump host httpforever.com and tcp port 80 -A -nn

完整文档 man tcpdump，下面只列高频项。

常用参数

参数作用
-i <iface>指定接口（不指定取第一个可用；-i any 监听所有）
-nn不解析主机名和端口名
-v / -vv / -vvv详细级别
-c <num>捕获指定数量后退出
-s <len>抓取长度（0 = 全包；新版默认 262144 bytes）
-w <file>写入 pcap 文件
-r <file>读取 pcap 文件离线解析
-A以 ASCII 打印 payload（截 HTTP：tcpdump -nSA port 80）
-X十六进制 + ASCII 打印（不要和 -A 一起用）
-S显示绝对序列号而不是相对编号
-tttt打印完整日期
-e显示以太网头（MAC 地址）

BPF 过滤语法

[proto] [src|dst] [host|net|port] <对象> [逻辑运算]

tcp and dst port 80 and src net 10.0.0.0/8
  • 协议：ip tcp udp icmp arp vlan ip6 …
  • 方向：src dst src or dst src and dst
  • 对象：host 10.0.0.1 / net 192.168.0.0/16 / port 443
  • 逻辑：and (&&) / or (||) / not (!) / ( )
sudo tcpdump -i en0 host 192.168.50.1          # 源或目的
sudo tcpdump -i en0 src host 192.168.50.1      # 源
sudo tcpdump -i en0 dst host 192.168.50.1      # 目的
sudo tcpdump -i en0 port 8080                  # 端口
sudo tcpdump -i en0 net 192.168                # 网段
sudo tcpdump -i en0 tcp                        # 协议

输出格式

14:24:12.196582 IP 192.168.91.29.56930 > 146.190.62.39.http: Flags [SEW], seq 193726297, win 65535, options [...], length 0

时间戳 / 协议层 / 源.端口 > 目标.端口 / TCP 标志位 / seq ack win 等字段 / 报文长度。

TCP Flags

TCP Flag缩写含义
SYNS会话建立请求
ACKA确认发送方的数据
FINF终止标识
RESETR立即中止连接
PUSHP立即推送数据
URGENTU优先于其他数据
NONE.占位符，通常用于 ACK

参考 tcpdump Flags。

接口列表

sudo tcpdump -D
# 1.en0 [Up, Running]
# 2.p2p0 [Up, Running]
# ...

macOS 下监听适配器必须 root。

实例

抓 eth1 上目的地址是 192.168.1.254 或 192.168.1.200、端口 80 的 TCP：

sudo tcpdump -i eth1 '((tcp) and (port 80) and ((dst host 192.168.1.254) or (dst host 192.168.1.200)))'

只抓 SYN：

sudo tcpdump -i eth1 'tcp[tcpflags] = tcp-syn'

抓 SYN + ACK：

sudo tcpdump -i eth1 'tcp[tcpflags] & tcp-syn != 0 and tcp[tcpflags] & tcp-ack != 0'

抓 DNS 请求：

sudo tcpdump -i en0 udp dst port 53

流量大时用 -c 限制数量：

sudo time tcpdump -nn -i en0 'tcp[tcpflags] = tcp-syn' -c 10000 > /dev/null

实时抓 8000 端口的 GET 包写入文件：

sudo tcpdump -i eth0 '((port 8000) and (tcp[(tcp[12]>>2):4]=0x47455420))' -nnAl -w /tmp/GET.log

MySQL Illegal mix of collations

Illegal mix of collations (utf8mb4_general_ci,IMPLICIT) and (utf8mb4_unicode_ci,IMPLICIT) for operation

utf8mb4_general_ci 和 utf8mb4_unicode_ci 两列直接做 JOIN / 比较时报错——MySQL 不会自动转换 collation。

解决

根治：统一字段 collation。MySQL 8.0+ 默认 utf8mb4_0900_ai_ci（基于 Unicode 9.0），5.7 默认 utf8mb4_general_ci，老库迁 8.0 时 collation 不会自动改，要小心遗留表。

临时绕：SQL 里 COLLATE 显式转：

SELECT * FROM a JOIN b ON a.name = b.name COLLATE utf8mb4_unicode_ci;

三种 collation

排序依据性能用途
utf8mb4_general_ci简化 ASCII 规则最快5.7 老库默认，新库不推荐
utf8mb4_unicode_ciUnicode 4.0略慢5.7 推荐
utf8mb4_0900_ai_ciUnicode 9.0≈ unicode_ci8.0+ 默认

性能差距在绝大多数业务里可忽略，同库统一比选哪个更重要。

参考 Illegal mix of collations | 少年阿斌。

按作者统计 git 贡献量

只看 commit 次数（git 内置，0 依赖）：

git shortlog -sn --no-merges

要带行数的版本，用 --numstat 给出每文件 +N -N filename，再用 --pretty=format 把作者名作定界符让 awk 聚合：

git log --numstat --pretty='COMMIT %aN' \
  | awk '/^COMMIT / { a = substr($0, 8); next }
         /^[0-9]/   { ins[a] += $1; del[a] += $2 }
         END        { for (k in ins) printf "%s\t+%d -%d\n", k, ins[k], del[k] }'

%aN 会走 .mailmap（同一人不同邮箱自动合并）。^[0-9] 自动跳过 binary 文件（numstat 对 binary 输出 - - file）。加 --author="..." 限单人；加 --since="2024-01-01" 限时段。

行数易被 reformatting / rename / 生成代码放大，跟 commit 数对照看比单看任一项准。

CRLF will be replaced by LF

$ git add .
warning: CRLF will be replaced by LF in X.

文件含 CRLF 行尾，git 按当前 core.autocrlf / .gitattributes 配置正在把它改成 LF。只是 warning 不是 error，提交不会被拦。

跨平台团队（Mac/Linux + Windows）想把行尾问题从 PR review 里彻底抹掉，仓库根放一份 .gitattributes：

* text=auto
*.sh text eol=lf
*.bat text eol=crlf

老仓库加这份配置后，跑 git add --renormalize . && git commit 让已入库的文件按新规则重写一遍——否则规则只对未来的修改生效。

* text=auto 一行覆盖 90% 场景。*.sh text eol=lf 是经典救场——Windows 同事 checkout 后行尾被换成 CRLF，bash deploy.sh 直接报 bad interpreter；*.bat 是对称情况。完整规则见 GitHub Docs: Configuring Git to handle line endings。

PHP strpos 找到时返回 0

strpos($haystack, $needle): int|false 找不到时返回 false，找到时返回位置 int——位置 0 在 if 里跟 false 等价，会被吞掉。

$s = 'abc';

if (strpos($s, 'a')) {            // 不输出，0 被当 false
    echo 'found';
}

if (strpos($s, 'a') !== false) {  // 正解
    echo 'found';
}

PHP 8.0+ 直接用 str_contains($haystack, $needle): bool，没有这个坑。

GitHub README 图片报 Content length exceeded

GitHub 把 README / issue / wiki 里所有外链图都走 Camo 匿名代理（隐藏访问者 IP 和 referer），代理默认 5 MB 上限。超过的图加载失败，alt 文字被替换成 Content length exceeded。

content_length_limit = parseInt(process.env.CAMO_LENGTH_LIMIT || 5242880, 10)

this / self / parent / static 在 PHP 类内部的区别

类内部访问成员有 4 个关键字，差别在「指向哪个类」与「是否跟着子类走」（多态 / late static binding）：

关键字指向LSB静态方法可用
self::写它的那个类（词法）否✓
parent::self 的父类否✓
static::实际调用时的类（late static binding，PHP 5.3+）✓✓
$this->当前实例所属的类✓✗（静态方法里无 $this）

self:: 和 static:: 的差别只在被子类继承后才显现：

class A
{
    public static function newSelf()   { return new self();   }
    public static function newStatic() { return new static(); }
}

class B extends A {}

var_dump(B::newSelf());   // object(A) — self 锁死在写它的 A 里
var_dump(B::newStatic()); // object(B) — static 跟着调用方

口诀：工厂方法要返回子类型 → static::；要锁定当前类不被覆盖 → self::。

2017-04-21 #php

CGI vs FastCGI vs PHP-CGI vs PHP-FPM

PHP 处理 web 请求要解决两件事——和 web server 通信的协议、PHP 自己的进程管理。这四个名词分两组踩在这两件事上：协议层是 CGI / FastCGI，PHP 实现是 PHP-CGI / PHP-FPM。

协议层

CGIFastCGI
进程模型每个请求 fork-and-exec 一次master + 长驻 worker pool
启动开销每次都加载解释器解释器常驻内存
数据流环境变量 + stdin/stdout二进制协议复用 socket 连接
现状历史遗留现代 web 标准

CGI 慢在反复 fork 和解释器反复初始化。FastCGI 把解释器留在内存，请求过来直接派给空闲 worker。

PHP 实现

PHP-CGIPHP-FPM
性质PHP 的 CGI binaryPHP 官方 FastCGI 进程管理器
进程管理无master + 动态 worker pool
改 php.ini必须杀进程重启，PHP 会停服kill -USR2 平滑 reload，请求不中断
现状淘汰主流（Laravel / WordPress / 大多数 PHP 栈）

PHP-FPM = PHP 解释器 + FastCGI 协议实现 + 进程管理。三件事的合体，不是单独某个组件。

实战：nginx + php-fpm

nginx 把 .php 请求按 FastCGI 协议丢给 php-fpm 的 socket：

location ~ \.php$ {
    fastcgi_pass   unix:/run/php/php8.2-fpm.sock;
    fastcgi_index  index.php;
    fastcgi_param  SCRIPT_FILENAME $document_root$fastcgi_script_name;
    include        fastcgi_params;
}

php-fpm 那头按 pool 配置 worker 数（pm.max_children 等），nginx 不感知 PHP 进程。

怎么选

现代栈一律 nginx + php-fpm，CGI / PHP-CGI 了解概念即可，生产不用。

PHP empty('0.0') 不是空

PHP 的 empty() 把 '0'（字符串零）当空，但 '0.0'（字符串小数零）当非空。DB 里把 decimal 字段存成 string 时会咬人——查到 '0.00' 误以为有值。

empty('0');           // true
empty('0.0');         // false ← 反直觉
empty((float) '0.0'); // true

判断数值前先 cast：empty((float) $value)。
