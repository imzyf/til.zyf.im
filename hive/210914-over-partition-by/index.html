Hive OVER (PARTITION BY) 窗口函数

2021-09-14 #hive

业务上要算”当年累计 income”这种基于组的滚动指标，普通 SUM(income) GROUP BY ... 一组只剩一行，行级累计要靠开窗函数 OVER (PARTITION BY ...)：每行都返回一个基于”窗口”内的聚合值，原始行数不变。

CREATE TABLE default.test_over_partition (
  fdate     Date,
  year      Int,
  month     Int,
  category1 String,
  category2 String,
  income    Double
);

INSERT INTO default.test_over_partition VALUES
  ('2020-01-01', 2020, 1, '3C', '电脑', 1010),
  ('2020-01-01', 2020, 1, '3C', '手机', 1011),
  ('2020-02-01', 2020, 2, '3C', '电脑', 1012),
  ('2020-02-01', 2020, 2, '3C', '手机', 1013),
  ('2020-03-01', 2020, 3, '3C', '电脑', 1014),
  ('2020-03-01', 2020, 3, '3C', '手机', 1015),
  ('2021-04-01', 2021, 4, '3C', '电脑', 1016),
  ('2021-04-01', 2021, 4, '3C', '手机', 1017),
  ('2021-05-01', 2021, 5, '3C', '电脑', 1018),
  ('2021-05-01', 2021, 5, '3C', '手机', 1019);

-- 按 year × category 分组，按 fdate 排序，行级累计 income
SELECT fdate, year, category2, income,
       SUM(income) OVER (PARTITION BY year, category1, category2 ORDER BY fdate) AS ttl_year_income
FROM default.test_over_partition;

PARTITION BY 决定窗口边界（每个 year × category 一个窗口），ORDER BY 决定窗口内累计的方向。换 AVG / RANK / ROW_NUMBER 等其他分析函数，套路一样。
